Fatch join 은 엔티티를 조회할때 연관관계 매핑된 모든 엔티티를 SQL join 을 통해서 한번에 가져오는 방식 입니다. 장점으론 n + 1 문제가 발생하지 않고, 단점으로는 모든 연관된 엔티티를 불러오기 때문에 불러올 때 많은 부화가 생기고, 비즈니스 로직상 사용하지 않는 데이터도 가져올 수 있다는 점이 있습니다.

Lazy loading 은 연관관계에 매핑된 엔티티를 . 연산자로 접근할때 SQL select 를 통해 가져오는 방식입니다. 장점으론 데이터가 실재로 사용할때만 데이터를 가져올 수 있고, batch size 를 조절하여 불러올 양을 조절할 수 있습니다 이로 인해 한번에 많은 부화가 DB 와 네트워크에 걸리는 것을 방지 할 수 있습니다. 단점으론 n+1 문제가 발생할 수 있다는 문제가 있습니다.
