
FilterRegistratoinBean 방식은 new 연산자를 사용하여 필터를 하나씩 서블릿에 추가하는 방식입니다. 스프링 빈이 아님으로 그 필터는 DI 를 스프링에서 제공하지 않고, 테스트가 어렵습니다. 

DelegatingFilterProxy 는 스프링 시큐리티에서 사용하는 방식으로 프록시 필터를 서블릿에 추가한 다음 프록시 필터가 filterChainProxy 를 거쳐 다른 필터체인들을 호출하는 방식으로 작동합니다. 여러 필터들을 필터 체인 형태로 등록할 수 있고, 스프링 빈으로 등록된 필터들을 등록하기 때문에 DI가 가능하고 테스트가 쉽습니다. 조건이나 api 엔드포인트에 따라 다른 필터 체인을 선택하여 사용하는 기능도 있습니다.
