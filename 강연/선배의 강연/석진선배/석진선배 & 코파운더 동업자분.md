


창업 계기
* 대표님이랑 친구여서 같이 창업함. 
* 프젝 참여제안 당시 코딩 6개월차 (그게 창업인지 몰랐음)
* 야구 데이터 관련 플렛폼 프젝이였음  포폴용으로 써야지 하는 마음으로 참여



## 1번쨰 프젝
대표님 LCK PD 출신 
창업 : 자기가 겪은 문제를 해결하는 방법 -> 스크림 데이터를 모아서 통계보여주는 사이트 만듬.

커피숍에서 시작한 프젝 -> 대표님이 사무실 구하기 시작하더니 그게 창업이 됨.

피보팅 8번

나온 결과물이 실패함 

이유 : 제품에 리텐션이 없었음.

리텐션 5% 도 안나옴 -> 거의 없었음.
그래서 할 마음도 안생김 -> 접음


## 2번째 프젝
유저 매칭 서비스 -> 이것도 리텐션 안나옴 
안나오는 이유 : 매칭해도 디코로 떠남.


우리도 음성 쳇 만들까? -> 이미 시장 점령하고 있는 디코가 있었음. 스타트업에서 하면 안되는 생각


## 3번째 프젝
화면 인터뷰를 하는 프젝 만들자

플레이 주요 장면들을 녹화해서 클리핑 해주는 서비스를 만들자 -> 이미 비슷한 서비스들이 많았음. 

베타버전 폴더 열기, 내보니기 2개 밖에 없었음. 
리텐션 50% 나옴.




개발자가 개발만 해야지 라고 생각하면 창업을 못했을꺼다.

직접 발로 뛰어서 홍보함 자연스럽게(그냥 유명 게시판에 글 올려보거나 하는등.)

1인 창업시 문제점
* 컨텍스트 스위칭이 심해짐
ai 를 사용해서 다 하더라도 개발하다가 마케팅하다가 비즈니스 하다가를 해야되기 때문에..








제품 만들고 싶을때
* 먼저 제품을 만들지 마라 or ai 로 데모만 만들던가해라
이 제품이 해결하는게 나의 문제인지 다른사람들도 있는 문제인지.

코파운더 님 경우 그냥 렌딩페이지만 만들고 사람들한테 수요 물어봄(롤갤, 인벤같은 게시판 등.)

직접 ~ 이거 쓰실거에요? 같은거 물어보면 대부분 쓸거라고 함 (별로 변별력이 없음.)
간접적으로 취미활동 뭐하는지, 뭘 좋아하는지 같이 물어보기

ai 로 생산성이 커짐으로 인해 회사에서 적은 인원으로 큰 프젝을 하고싶어 함 하나의 사람이 많은 책임을 자실 수 있음. -> 그당시에 왜 그 선택을 했는지를 말할 수 있어야 함. (순수히 궁금해서 왜 그랬는지)




제품을 만들때 중요하게 생각하는거

처음 만들땐 유저 니즈 파악하는게 중요
내가 만들고 싶어하는거는 안만들었으면 좋겠음 -> 유저가 원하는 제품을 만들어야함
VOC 가 없다면 직관에 의존해야함 만들면서 VOC 해보셈

나의 주관을 기준으로 가설을 새워야 하지만, 실제 검증은 유저가 해야한다.
-> 뼈대는 나의 생각 지지대는 유저의 인터뷰등 유저 생각 (답은 유저한테)



반응 2가지가 있음
1. 일단 무조건 제품 런칭해서 사용자 반응 봐라
2. 사용자가 필요한걸 만들어라

잘 판단해바야 할듯
일단 후자가 좀더 끌리긴 함.

일단 시작을 하고 스케일링 할 수 있으니 그건 이따가 생각해라

10~100명의 너의 제품을 사랑할 유저를 찾아라
